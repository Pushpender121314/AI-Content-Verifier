# 📝 AI Content Verifier: Stylistic Text Forensic Hub

An end-to-end Natural Language Processing (NLP) framework built to distinguish between human-authored essays/articles and content generated by Large Language Models (LLMs). This project extracts structural text features and phrasal patterns using word N-grams, trains a **Logistic Regression Classifier**, and serves instant forensic results through an interactive **Streamlit web application**.

---

## 📌 Project Architecture & Pipeline

The project development workflow handles the complete transformation from raw text blocks into real-time probability distributions:

1. **Analytical Feature Exploration (`ai-vs-human-text-detection.ipynb`)**
   * Processes large-scale corpora to observe variance metrics between human strings and machine outputs.
   * Visualizes stylistic data distributions, phrasing density distributions, and vocabulary repetition parameters.

2. **Core Text Forensics Pipeline (`train.py`)**
   * **Missing Feature Sanitization:** Cleans the inputs by dropping any null string rows to avoid computational errors.
   * **Downsampling Check:** Includes automated data parsing thresholds. If the dataset exceeds 40,000 items, it automatically samples a balanced subset to optimize local computing speed without degrading model accuracy.
   * **TF-IDF N-gram Extraction:** Uses a `TfidfVectorizer` configured with `ngram_range=(1, 2)` (extracting both single words and consecutive word pairs) capped at 10,000 maximum attributes to capture repetitive writing structures typical of LLMs.
   * **Classifier Training:** Fits a highly tuned `LogisticRegression` model optimized with `C=5.0` to maximize boundary classification scores on linguistic features.
   * **Asset Serialization:** Exports the final engine binaries into `ai_detector_model.pkl` and `text_vectorizer.pkl`.

3. **Live Web Application Interface (`app.py`)**
   * Built entirely using **Streamlit** to offer a responsive text submission UI.
   * Unpacks saved vector weights to reconstruct the original processing matrix space.
   * Outputs real-time dual probability bars detailing the precise percentages of Human Authorship vs. LLM Generation.

## 🛠️ Step-by-Step Environment Setup Guide
Follow these instructions to activate your project environment and execute the pipeline cleanly.

**Step 1: Open Terminal & Navigate:**
Open your Terminal (macOS/Linux) or Command Prompt/PowerShell (Windows) and move into your summer training project folder:

      cd path/to/your/project-folder

**Step 2: Establish a Virtual Environment:**
Setting up a separate virtual environment keeps your project isolated so that visual libraries, Streamlit, and Scikit-Learn don't cause version conflicts with other code on your computer.

# Create the virtual environment named 'forensics_env'
      python -m venv forensics_env

# Activate the environment:
# On Windows (Command Prompt):
      forensics_env\Scripts\activate
# On Windows (PowerShell):
      .\forensics_env\Scripts\Activate.ps1

# On macOS / Linux:
source forensics_env/bin/activate

**Step 3: Install Project Dependencies**
Run the install command to populate all necessary text processing and application packages:


      pip install --upgrade pip
      pip install -r requirements.txt

**Step 4: Position Your Source Data**
Ensure your source dataset is saved as ai_vs_human_text.csv and placed inside a folder named data/ sitting inside your main root folder.

## 🚀 Operational & Training Guide
**1. Run the Training Engine:**
Before running your web dashboard application, the model needs to build its vocabulary matrix and learn structural text rules. Run this command from your terminal:

      python train.py

This handles data processing, samples inputs to match local hardware capabilities, and saves the ai_detector_model.pkl and text_vectorizer.pkl files.

**2. Boot the Interactive Web UI Dashboard:**
Once the files are saved in your folder, initiate your frontend dashboard using:

      streamlit run app.py
Streamlit will spin up local hosting channels and automatically open a fresh UI dashboard tab inside your web browser (usually at http://localhost:8501).

**🧐 Expected Execution Terminal Log:**
When executing python train.py, your console will display tracking indicators followed by an evaluation metrics matrix:

# 🛡️ Interactive User Simulation Testing
You can use the large text entry canvas in the live Streamlit Web Dashboard to demonstrate how the system works to reviewers or evaluators:

**Human Authorship Test:**

 Paste text with standard vocabulary pacing variance and organic cadence, such as: "Honestly, looking back at my childhood, we spent almost every evening outside until the streetlights came on. It wasn't anything fancy, just standard neighborhood games, but those moments really shaped how I connect with people today." The engine will return a ✅ Content Verified: High Probability of Human Authorship! badge.

**AI Generation Test:**

 Paste highly predictable or rigid structures, such as: "In conclusion, it is important to note that Large Language Models possess a wide array of capabilities. Furthermore, they optimize operational efficiencies across multiple sectors, thereby transforming modern digital communication." The model will instantly trigger a flashing 🚨 CRITICAL WARNING: High Probability of AI-Generated Content! alert panel.
